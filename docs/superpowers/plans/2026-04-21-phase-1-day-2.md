# Phase 1 Day 2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Finish the Foundation phase — expand Prisma schema to the full data model, migrate Neon, seed demo data, lock down the dev endpoint in production, and tag `foundation-working` so Phase 2 (message ingestion) can start on a solid floor.

**Architecture:** Single-service inline-worker Render deploy (per prior decision — web + BullMQ worker in one Node process via `RUN_WORKER_INLINE`). Prisma 7 with pg adapter, Neon pooled URL for the app runtime and Neon direct URL for migrations. All tables include `coachId` for multi-tenant readiness. Seed script is idempotent and safe to re-run.

**Tech Stack:** NestJS 11, Prisma 7 + `@prisma/adapter-pg`, Neon Postgres, BullMQ 5 on Redis, pnpm workspaces, Render.

## Task 5: Seed script (1 coach, 5 parents, 5 kids, 10 sessions)

**Files:**

- [ ] **Step 1: Create backend/prisma/seed.ts**

```ts
import 'dotenv/config';
import { PrismaClient } from '@prisma/client';
import { PrismaPg } from '@prisma/adapter-pg';
import { Pool } from 'pg';

const DEMO_COACH_PHONE = '+15555550100';

const PARENTS = [
  { name: 'Alice Chen',    phone: '+15555550101', kid: { name: 'Priya Chen',    age: 11 } },
  { name: 'Ben Okafor',    phone: '+15555550102', kid: { name: 'Noah Okafor',   age: 9  } },
  { name: 'Carla Duarte',  phone: '+15555550103', kid: { name: 'Mateo Duarte',  age: 13 } },
  { name: 'Divya Patel',   phone: '+15555550104', kid: { name: 'Aarav Patel',   age: 10 } },
  { name: 'Elena Rossi',   phone: '+15555550105', kid: { name: 'Luca Rossi',    age: 12 } },
];

async function main() {
  const connectionString = process.env.DATABASE_URL;
  if (!connectionString) throw new Error('DATABASE_URL is not set');

  const pool = new Pool({ connectionString });
  const adapter = new PrismaPg(pool);
  const prisma = new PrismaClient({ adapter });

  const coach = await prisma.coach.upsert({
    where: { id: 'demo-coach' },
    create: {
      id: 'demo-coach',
      name: 'Coach Demo',
      phone: DEMO_COACH_PHONE,
      timezone: 'America/Los_Angeles',
    },
    update: {},
  });

  for (const p of PARENTS) {
    const parent = await prisma.parent.upsert({
      where: { coachId_phone: { coachId: coach.id, phone: p.phone } },
      create: {
        coachId: coach.id,
        name: p.name,
        phone: p.phone,
        preferredChannel: 'SMS',
      },
      update: {},
    });

    await prisma.kid.upsert({
      where: { id: `kid-${parent.id}` },
      create: {
        id: `kid-${parent.id}`,
        coachId: coach.id,
        parentId: parent.id,
        name: p.kid.name,
        age: p.kid.age,
      },
      update: {},
    });
  }

  const kids = await prisma.kid.findMany({ where: { coachId: coach.id }, orderBy: { createdAt: 'asc' } });
  const now = new Date();

  // 10 sessions: 5 completed in the past, 5 upcoming. Deterministic ids so re-seeding is idempotent.
  const plan = [
    ...kids.map((k, i) => ({ kidId: k.id, offsetDays: -(i + 1) * 3, status: 'COMPLETED' as const, paid: true })),
    ...kids.map((k, i) => ({ kidId: k.id, offsetDays: (i + 1) * 2,  status: 'CONFIRMED' as const, paid: false })),
  ];

  for (const [i, s] of plan.entries()) {
    const scheduledAt = new Date(now);
    scheduledAt.setDate(scheduledAt.getDate() + s.offsetDays);
    scheduledAt.setHours(16, 0, 0, 0);

    await prisma.session.upsert({
      where: { id: `session-${i}` },
      create: {
        id: `session-${i}`,
        coachId: coach.id,
        kidId: s.kidId,
        scheduledAt,
        durationMinutes: 60,
        status: s.status,
        paid: s.paid,
        paymentMethod: s.paid ? 'CASH' : null,
      },
      update: {},
    });
  }

  const counts = {
    coaches: await prisma.coach.count(),
    parents: await prisma.parent.count(),
    kids: await prisma.kid.count(),
    sessions: await prisma.session.count(),
  };

  console.log('Seed complete:', counts);
  await prisma.$disconnect();
  await pool.end();
}

main().catch(async (err) => {
  console.error(err);
  process.exit(1);
});
```

    process.env.NODE_ENV = 'development';
    const ctrl = await makeController();

    expect(res.ok).toBe(true);

## Self-review notes

- Every Day 2 bullet from `context.md` is covered: prod smoke (Task 1 — adapted to single-service since we chose Option A), full schema (Tasks 2–4), seed (Task 5), dev endpoint guard (Task 6), tag (Task 7).
- `DIRECT_URL` added explicitly because `prisma.config.ts` currently only reads `DATABASE_URL`, and Neon migrations need the direct URL to avoid pgbouncer's prepared-statement issues.
- Seed uses deterministic ids (`kid-<parentId>`, `session-0`…`session-9`) so re-running is safe — matches the "idempotent" requirement and avoids a `deleteMany` hammer that would nuke demo state mid-build.
- Tests are scoped to what's testable without standing up more plumbing: the dev-guard test is isolated unit-style; schema/seed verification is manual (counts + `\dt`), which is appropriate for one-shot infra work.
